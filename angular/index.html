Introduction

As you know, at jtsec we love Pringles! In fact, we have a whole shelf full of cans from all over the world. Every time someone goes on a trip to a foreign country, they try to bring a can that we don’t have in order to add it to the collection. The problem is that it is getting harder and harder to know which cans we already have in the office and we have to keep calling each other to see if we already have it or if it is a new one.

So, to test your skills, we propose you to develop a web application to make an inventory of the Pringles we already have on the shelf. The application must meet the following requirements.

Environment:

  • Must be developed in Linux
  • Docker and docker-compose must be installed.
  • Your browser of choice.
  • Your text editor of choice.

Technologies:

  • Relational Database (MySQL): Stores the data the application needs to work.
  • Backend (Laravel 5): API to interact with the database.
  • Frontend (AngularJS): View to interact with the API.
  • Style (Bootstrap): The view must use Bootstrap.
  • Execution in Docker: The application will be ready to run in containers, so that it runs in the same conditions for everyone.

User stories:

  • Anyone will be able to add a can of Pringles to the database.
  • Anyone can add the following information about a can: name, main color, nationality, when in was added, who brought it.
  • Anyone can remove a can.
  • Anyone will be able to visualize the cans with all the information that we have about them. In addition, the color of the can will also have to be indicated in some way visually. This means that if the can is red, the text should appear in the same color.
  • (Optional) Anyone can edit a can that has been added previously.

Starting point

To facilitate the development, a template is provided from which you can start. It is included:

  • Definition of the execution environment with docker-compose. It is not necessary any kind of modification.
  • As an example, we have developed the backend, frontend and table of the database to store and interact with “users”. The only purpose of a user is to use it to indicate who has brought a can.

Angular

  • The root directory is “angular”
  • The users’ view is in “angular/users.html”
  • The whole AngularJS definition is in “angular/scripts/pringlesapp.js”

Laravel

  • The database migrations are defined in “database/migrations”
  • The User model is defined in “app/User.php”
  • The API routes and their functionality are defined in “app/Http/routes.php”

Tasks

The tasks to be performed are:

  1. Through a migration in Laravel, create a table to be able to store the information related to the cans (name, main color, nationality, date added and who brought it). This last field must be related to the “users” table already created.
  2. Define the routes in Laravel to add and remove cans with the API.
  3. Define the factory in Angular to be able to consume the developed API.
  4. Define the controller with the necessary logic and add the view in Angular to be able to use the API from the web interface.
  5. (Optional) Define the path in Laravel to edit cans with the API.
  6. (Optional) Add this endpoint to the Angular factory.
  7. (Optional) Add to the controller the necessary logic and modify the Angular view to make use of the new functionality from the web interface.
  8. Write documentation in English explaining what has been done, decisions made, design of the database… with screenshots of the UI (PDF or Markdown).

Extra:

  • We ask that good practices be followed in the development of the test.
  • Optionally, use git for version control (locally). The project is already delivered with a “First commit”.
  • You are free to develop the tasks as you consider best as long as the requirements are met. When in doubt, make a decision and document why you have chosen that path.

Resources

This is a basic test to check the developer’s skills in handling new technologies. Therefore, it is understood that this may be the first time that you work with Angular or Laravel. That is why the following resources are recommended if you consider you are a novice in these technologies:

  • Google and Stackoverflow.
  • The Laravel 5 documentation (https://laravel.com/docs/5.1).

Execution

Finally, to run the project, just execute the following command:

docker-compose up -d

Services:

Frontend (APP): http://localhost:8081
Backend (API): http://localhost:8080
Database: 127.0.0.1:3306

In addition, if this is the first time you run it (or have created a new migration), you should apply the changes as follows:

docker-compose exec app php artisan migrate

It is also possible to connect directly to the database to check that the tables have been created successfully. For example, to do this with the mysql tool:

mysql -h 127.0.0.1 -u root -p laravel

The root password is: secret

Note that although the connection is to localhost, the IP must be specified to avoid the Linux socket connection. We need it to connect to the running container that is exposing a port, not the host.
